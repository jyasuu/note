# 🟦 1. Create an Elasticsearch Index

Indexes are for manually-managed data (no automatic rollover).

### ➤ Create an index

```bash
curl -X PUT "http://localhost:9200/my-index" \
  -H 'Content-Type: application/json' \
  -d '{
    "settings": {
      "number_of_shards": 1,
      "number_of_replicas": 1
    },
    "mappings": {
      "properties": {
        "timestamp": { "type": "date" },
        "message":   { "type": "text" },
        "host":      { "type": "keyword" }
      }
    }
  }'
```

***

# 🟩 2. Insert a Document into the Index

### ➤ Add a document (auto-generated ID)

```bash
curl -X POST "http://localhost:9200/my-index/_doc" \
  -H 'Content-Type: application/json' \
  -d '{
    "timestamp": "2026-03-18T10:00:00Z",
    "message":   "System startup complete",
    "host":      "server-001"
  }'
```

### ➤ Add a document with custom ID

```bash
curl -X PUT "http://localhost:9200/my-index/_doc/1" \
  -H 'Content-Type: application/json' \
  -d '{
    "timestamp": "2026-03-18T10:00:00Z",
    "message":   "Custom ID example",
    "host":      "server-001"
  }'
```

# 🟧 3. Create a Data Stream

Data streams are designed for **time-series log or metric data** and are automatically managed (rollover + backing indices).

Creating a data stream requires:

✔ Index Template (with data\_stream enabled)  
✔ Ingesting a document with `@timestamp`

***

## 🥇 Step 1 — Create an Index Template for Data Stream

```bash
curl -X PUT "http://localhost:9200/_index_template/logs-template" \
  -H 'Content-Type: application/json' \
  -d '{
    "index_patterns": ["logs-*"],
    "data_stream": {},
    "template": {
      "mappings": {
        "properties": {
          "@timestamp": { "type": "date" },
          "message":    { "type": "text" },
          "level":      { "type": "keyword" },
          "host":       { "type": "keyword" }
        }
      }
    }
  }'
```

***

## 🥈 Step 2 — Create the Data Stream

```bash
curl -X PUT "http://localhost:9200/_data_stream/logs-app"
```

***

## 🥉 Step 3 — Ingest a Document into the Data Stream

⚠ Data streams **require `@timestamp`**.

```bash
curl -X POST "http://localhost:9200/logs-app/_doc" \
  -H 'Content-Type: application/json' \
  -d '{
    "@timestamp": "2026-03-18T10:20:00Z",
    "message": "User login success",
    "level":   "INFO",
    "host":    "server-002"
  }'
```

***

# 2️⃣ Read Operations (R of CRUD)

## ✔ Get a document

```bash

```

```bash

{
  "docs": [
    { "_index": "my-index", "_id": "1001" },
    { "_index": "my-index", "_id": "1002" }
  ]
}
```

## ✔ Search (match)

```bash
GET my-index/_search
{
  "query": {
    "match": {
      "message": "error"
    }
  }
}
```

## ✔ Search (filter by date)

```bash

{
  "query": {
    "range": {
      "timestamp": {
        "gte": "now-1d"

    }
  }
}
```

# 🟩 Index Management (settings, mapping, alias)

```bash

{
```

```bash
PUT logs-2026/_mapping
```

```bash
POST _aliases

  "actions": [
```

# 🟨 ILM (Index Lifecycle Management)

## ✔ Create ILM policy (rollover + delete)

```bash
PUT _ilm/policy/my-ilm
{
  "policy": {
    "phases": {
      "hot": {
        "actions": {
          "rollover": { "max_size": "1gb" }
        }
      },
      "delete": {
        "min_age": "30d",
        "actions": { "delete": {} }
      }
    }
  }
}
```

## ✔ Apply ILM to index template

```bash
PUT _index_template/my-template
{
  "index_patterns": ["logs-*"],
  "template": {
    "settings": {
      "index.lifecycle.name": "my-ilm",
      "index.lifecycle.rollover_alias": "logs"
    }
  }
}
```

# 🟪 Ingest Pipelines (Log Parsing, Transformations)

## ✔ Create a pipeline

```bash
PUT _ingest/pipeline/process-logs
{
  "processors": [
    {
      "grok": {
        "field": "message",
        "patterns": ["%{TIMESTAMP_ISO8601:ts} %{LOGLEVEL:level} %{GREEDYDATA:msg}"]
      }
    }
  ]
}
```

## ✔ Index with pipeline

```bash
POST my-index/_doc?pipeline=process-logs
{
  "message": "2026-03-18T10:00:00Z INFO Startup complete"
}
```
